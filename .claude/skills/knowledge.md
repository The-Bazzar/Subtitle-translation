# 字幕术语知识库 + 第三次校对

浏览已听录的英文字幕和中文翻译字幕，建立术语知识库，然后基于知识库对中文翻译进行第三次校对。

## 触发条件

用户提供视频文件夹路径或直接说"建立知识库"、"校对翻译"、"knowledge"。

## 工作流程

### 阶段 1：浏览字幕，建立知识库

同时阅读英文 `.srt`（唯一原文依据）和中文 `.zh.srt`（仅作当前翻译参考），抽取以下四类术语和表达：

1. **核心主题词汇**：紧紧围绕**该视频**主题的关键词和句式。不要带入任何预设——完全根据当前视频的实际内容判断。

2. **专业术语/学术概念**：需要在整个项目中统一理解的词汇。只抽取**该视频实际出现**的术语，不要推测或扩展。

3. **作者态度基调**：体现作者对特定客体的感情倾向和态度。关注语气词、程度副词、反讽、夸张等修辞手法。**必须结合上下文**判断，同一词汇在不同语境下可能表达不同态度。

4. **核心观点表达**：全文反复出现的核心论点或关键表述。注意前后呼应、递进关系。

**重要原则**：
- 以英文 `.srt` 为唯一原文依据，中文 `.zh.srt` 仅作翻译参考（最终会被校对修改）
- **时刻关注上下文**：前后字幕之间的逻辑关系比单句更重要
- **避免自相矛盾**：同一术语在不同语境下如果含义不同，需分别标注语境
- **不要套用其他视频的知识**：每个视频有独立的知识库

### 阶段 2：输出知识库

在当前视频文件夹下建立 `glossary.md` 文件。格式参考（**仅为格式示例，内容需根据视频填充，以下术语不代表任何真实视频**）：

```markdown
# 术语知识库 — <视频标题>

## 核心主题词汇
| 英文 | 中文 | 备注 |
|------|------|------|
| EXAMPLE_TERM | 示例翻译 | 此行为格式参考，实际内容根据视频提取 |

## 专业术语
| 英文 | 中文 | 上下文说明 |
|------|------|-----------|
| EXAMPLE_TERM | 示例术语 | 在第X条字幕中首次出现，第Y条中再次使用 |

## 作者态度基调
- [态度描述 — 根据本视频实际内容判断]

## 核心观点
1. [核心论点 — 从本视频提取]
```

> **注意**：以上 markdown 表格中的 `EXAMPLE_TERM` 仅为格式示例，实际填写时必须替换为当前视频字幕中真实出现的术语。**每个视频的知识库内容独立，不可跨视频复用。**

### 阶段 3：基于知识库校对翻译

用知识库内容作为系统提示词的补充，对 `.zh.srt` 进行第三次校对：

1. 将 `glossary.md` 的内容附在校对 system prompt 之后，格式：`"以下是本视频的术语知识库，请严格遵循其中的翻译规范：\n{glossary_content}"`
2. 调用 `proofread_subtitles()` 进行中英校对
3. 覆盖写入 `.zh.srt`
4. 重新合成 `.zh.ass` + `.zh-en.ass`

## 工具链

```
glossary.md (术语知识库)
     │
     ▼
translate_srt.py  (第三次校对，加载 glossary 到 prompt)
     │
     ▼
.zh.srt (final) → .zh.ass + .zh-en.ass
```

## 注意事项

- 知识库以英文原文为唯一标准，中文翻译可以且**应当被修正**
- 抽出术语后，用户可手动补充更正 `glossary.md`，后续校对准会生效
- 知识库不应超过 100 行，保持精炼
- Agent 读取字幕时应使用 Read 工具分批读取，避免超长文件撑爆上下文
